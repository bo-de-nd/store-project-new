# منصة عرض وطلب المنتجات عبر واتساب — متجر لمسة

مشروع كامل من جزأين:
- **backend**: خادم Node.js + Express + قاعدة بيانات SQLite (ملف واحد، بدون حاجة لإعداد سيرفر قاعدة بيانات منفصل)
- **frontend**: تطبيق React (Vite) يتصل بالباك إند عبر API

---

## 1) تشغيل الباك إند

```bash
cd backend
npm install
cp .env.example .env
```

افتح ملف `.env` وعدّل:
- `ADMIN_USERNAME` و `ADMIN_PASSWORD` — بيانات دخول لوحة التحكم
- `JWT_SECRET` — أي نص عشوائي طويل وسري
- `STORE_WHATSAPP` — رقم واتساب صاحبة المتجر بصيغة دولية بدون علامة + (مثال: 967777123456)
- `STORE_NAME` — اسم المتجر

ثم شغّل الخادم:
```bash
npm start
```

سيعمل على: `http://localhost:4000`
أول تشغيل ينشئ قاعدة البيانات (`backend/data/store.db`) تلقائيًا مع حساب الأدمن ومنتجات تجريبية.

---

## 2) تشغيل الفرونت إند

```bash
cd frontend
npm install
cp .env.example .env.local
```

تأكد أن `VITE_API_URL` في `.env.local` يشير لعنوان الباك إند (افتراضيًا `http://localhost:4000/api`).

```bash
npm run dev
```

سيعمل على: `http://localhost:5173`

---

## 3) تجربة المشروع

- **واجهة العميل**: تصفح، بحث، سلة، إتمام طلب → يفتح واتساب تلقائيًا برسالة فيها رقم الطلب.
- **طلباتي**: تتبع الطلبات عبر رقم الهاتف.
- **لوحة التحكم** (زر "دخول الإدارة"): سجّل دخول ببيانات `.env`، أضف/عدّل منتجات، حدّث حالة الطلبات، أدخل رقم الطلب لتأكيد استلام الدفع، واطّلع على الإحصائيات والفاتورة (PDF).

---

## ملاحظة: إصدار Node.js

تم تحديد إصدار Node.js (22.x) صراحة في `backend/package.json` (حقل `engines`) وفي `render.yaml`، لتفادي مشاكل البناء التي تظهر مع إصدارات Node الحديثة جدًا (مثل 24) خصوصًا مع مكتبة `better-sqlite3`. عند التشغيل المحلي، يُفضّل استخدام Node 22 LTS أيضًا لنفس السبب.

## 4) عند الاستضافة المجانية (الخطوة القادمة)

عند اختيار موقع استضافة، ستحتاج عادة لخدمتين منفصلتين:
1. استضافة الباك إند (Node.js) — مثل Render أو Railway.
2. استضافة الفرونت إند (ملفات ثابتة بعد `npm run build`) — مثل Vercel أو Netlify.

بعد النشر، حدّث:
- `FRONTEND_ORIGIN` في إعدادات الباك إند برابط الفرونت إند المنشور.
- `VITE_API_URL` في إعدادات الفرونت إند برابط الباك إند المنشور، ثم أعد البناء (`npm run build`).

⚠️ ملاحظة أمان: قبل النشر الفعلي، غيّر `ADMIN_PASSWORD` و `JWT_SECRET` لقيم قوية وغير افتراضية.
